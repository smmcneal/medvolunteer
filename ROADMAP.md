- ⚠️ **If you are an agent working through the Cowork sandbox: do NOT run `git add`/`git commit`, `npm run build`, `npm run lint`, or `tsc` through the Linux bash mount for this repo.** The mount desyncs from the Windows filesystem — it returns partial file contents (a 31-line file read back as 4 lines, cut mid-word) and serves stale `stat` data, which makes git skip its content check and report a genuinely modified file as `CLEAN`. A `git add -A` through it can stage a partial file or silently skip a changed one. The Read/Write/Edit file tools go to Windows directly and are accurate — use those, and have Sean build and commit on Windows.
